# Tattoo Artist Scarcity & Booking System Strategy

## Objective
Implement a high-conversion booking system that creates **perceived scarcity**, increases demand, improves client quality, and elevates brand positioning.

This system replaces "open booking" with **controlled access, booking drops, and application-based intake**.

---

# Core Strategy Principles

## 1. Controlled Access (No Open Booking)
- Eliminate "DM to book anytime"
- Replace with **scheduled booking drops**
- Limit booking windows to **24–48 hours max**

---

## 2. Perceived Scarcity
- Always display **less availability than actual**
- Use phrases like:
  - "Last 3 spots remaining"
  - "Only taking 5 projects this month"
- Never show full calendar availability

---

## 3. Authority Positioning
- Shift from:
  - "Book with me"
- To:
  - "Apply to work with me"

---

## 4. Selectivity
- Use **application-based booking**
- Accept only projects aligned with artist style
- Reject or defer low-quality requests

---

# System Architecture

## Booking Flow

Instagram → Website → Booking Form → Review → Acceptance → Deposit → Appointment

---

## Components to Build

### 1. Booking Status System
- Dynamic status display:
  - "Books Closed"
  - "Books Open"
  - "Limited Availability"
- Include:
  - Next booking drop date
  - Countdown timer (optional)

---

### 2. Booking Drop System
- Monthly or bi-monthly drops
- Open for limited time window
- Automatically close after window ends

---

### 3. Waitlist System
- Capture users when books are closed
- Store:
  - Name
  - Email
  - Tattoo idea
- Use for:
  - Filling cancellations
  - Early access offers

---

### 4. Application Form (Replace Direct Booking)

Fields:
- Full Name
- Email
- Phone
- Tattoo Description
- Placement
- Size
- Budget (optional but recommended)
- Reference Upload
- Availability Window

Behavior:
- Submit = "Request", NOT confirmed booking
- Show message:
  - "Your request has been received. Selected projects will be contacted."

---

### 5. Flash Drop System
- Create "Available Designs" section

Each card includes:
- Artwork image
- Price
- Placement suggestion
- "Claim Design" button

Rules:
- First come, first served
- One-time use only
- Remove once claimed

---

# UI / UX Requirements

## Website Messaging

### When Closed:
- "Books are currently closed"
- "Next booking drop: [DATE]"
- "Join the waitlist for priority access"

### When Open:
- "Books are now open for [MONTH]"
- "Limited spots available"

---

## Scarcity Messaging Rules

Always use:
- "Limited"
- "Only X spots"
- "Applications closing soon"

Never use:
- "I have openings"
- "Trying to fill spots"
- Discounts to fill time

---

# Instagram Integration Strategy

## Story Flow
1. Teaser:
   - "Books opening Sunday 6PM"
2. Reminder:
   - Countdown sticker
3. Launch:
   - "Books are LIVE"
4. Close:
   - "Books are now CLOSED"

---

## Post Examples

- "Last 2 spots left this month"
- "Books filled — thank you everyone"
- "Next drop coming soon"

---

## Cancellation Strategy
Instead of:
- "I have availability"

Use:
- "Had a cancellation — offering this spot to waitlist first"

---
